=== REDDIT POST ===
**I was so frustrated with manually tweaking CSS clip-path values that I built a drag-and-drop generator.**

As a frontend dev, I often need custom shapes for overlays, image masks, or UI elements. Writing polygon coordinates by hand is tedious—trial and error, refreshing, tweaking numbers, rinse and repeat. Existing tools felt either too simplistic or clunky, so I decided to scratch my own itch.

I built a visual editor where you can drag points directly on a canvas, adjust curves, and see the CSS update in real time. It supports polygons, circles, ellipses, and inset shapes, and you can export clean CSS code or copy it directly. It’s also a PWA, so it works offline.

**Tech side:** It’s a Next.js app with TypeScript, using Canvas for the interactive editor and Tailwind for UI. The trickiest part was making the drag interactions feel smooth while keeping the generated CSS accurate.

If you work with CSS clipping or need non-rectangular layouts, give it a try:  
[https://shipmicro.com/tools](https://shipmicro.com/tools)

I’d love brutal feedback—especially on UX, performance, or anything that feels off. What features would make this actually useful for your workflow?

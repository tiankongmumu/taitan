=== REDDIT POST ===
I was so frustrated with constantly overpaying for Ethereum gas fees, especially when doing batch transactions for my other projects. I’d check a tracker, send a tx, and then watch the price drop right after—super annoying. So I made a real-time gas tracker with a few extra things that helped me personally.

The main difference is that it uses a lightweight ML model to predict short-term gas trends instead of just showing the current price. It also supports multiple EVM chains (Polygon, Arbitrum, etc.) and has a simple transaction scheduler that lets you set a target gas price and get notified (or auto-submit) when it’s hit.

Tech-wise, it’s a Next.js frontend with a Go backend for the prediction service. The model is a simple LSTM trained on historical gas data, and we update it daily. The scheduler uses a queue system with Redis to monitor gas prices without hammering RPC endpoints.

If you want to check it out, it’s here:  
https://shipmicro.com/tools

I’d love some brutal feedback—especially if you’ve built something similar or if you think the prediction feature is actually useful vs. just a gimmick. Also, if there are other chains or features you’d want, let me know.

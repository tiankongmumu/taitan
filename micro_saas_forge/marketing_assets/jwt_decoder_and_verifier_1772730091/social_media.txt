=== REDDIT POST ===
I was so frustrated with how clunky and slow most JWT debuggers felt, especially when I was iterating on auth flows or debugging production issues. I’d have a token, paste it somewhere, wait for a decode, then maybe verify it—it always felt like just enough friction to break my flow.

So I made a JWT decoder and verifier that focuses on two things: an awesome, clean UI and zero perceived latency. I wanted it to feel instant—paste a token and see it decoded and validated in real time as you type or edit.

The frontend is Next.js with TypeScript and Tailwind, and the JWT parsing/verification runs entirely client-side (using `jsonwebtoken` in a Web Worker to keep the UI snappy). No tokens are sent to a server—everything happens in your browser. You can paste a token, edit the payload or signature directly in the UI to see how validation changes, and toggle between HS256 and RS256 to test different signing methods.

It’s been a handy little security workstation for my own projects, and I hope it can save some of you time too.

Check it out here:  
https://jwt-security-workstation-dlbam3h25-tiankongmumus-projects.vercel.app

I’d love brutal feedback—especially if you run into any issues with specific token formats, or if there are features you’ve wished for in a JWT tool that I haven’t thought of.

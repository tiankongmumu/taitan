=== REDDIT POST ===
**I was so frustrated with waiting for SQL query optimizers to run on remote servers, so I made one that runs entirely in your browser with zero latency.**

I work with a lot of legacy databases, and I’m constantly tweaking queries for performance. Every time I’d paste a query into an online optimizer, I’d have to wait for a round trip, deal with network lag, and sometimes worry about sending sensitive schemas over the wire. It felt unnecessary for something that should be almost instantaneous.

So I built **SQL Query Optimizer**, a tool that runs locally in your browser via WebAssembly. It takes your SQL query and schema, analyzes joins, indexes, and filters, and suggests optimizations—all without your data ever leaving your machine. The core engine is written in Rust, compiled to WASM, with a lightweight Next.js/TypeScript frontend. It’s basically a static site; there’s no backend processing at all.

I’ve been using it myself for a few months to clean up slow queries before deployment, and it’s saved me a ton of manual analysis time. It’s not a full replacement for a database’s `EXPLAIN` or a dedicated profiler, but it’s handy for quick, private feedback.

If you want to try it out:  
[https://shipmicro.com/tools/sql-opt](https://shipmicro.com/tools/sql-opt)

I’d love some **brutal feedback**—especially if you work with complex queries or unusual SQL dialects. Does the analysis feel accurate? Are the suggestions actually useful? Is there a feature you’d kill to have in a tool like this?

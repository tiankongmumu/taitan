=== REDDIT POST ===
**I was so frustrated with manually translating JSON structures into PostgreSQL schemas, so I made a tool that does it automatically—and keeps them in sync.**

I work on a lot of side projects and small APIs where I start with a JSON sample or an API response as my data model. Every time, I’d find myself writing the same CREATE TABLE statements by hand, tweaking data types, adding constraints, and then having to reverse-engineer SQL back to JSON for documentation or frontend modeling. It felt like busywork, and I kept making small mistakes with nullability or varchar lengths.

So I built a PostgreSQL schema generator that converts JSON to a fully typed SQL schema, but also goes a few steps further:

- **AI-assisted suggestions** – It analyzes your JSON and recommends optimal data types, indexes, and constraints (e.g., when to use `text` vs `varchar`, or where a foreign key might make sense).
- **Bi-directional sync** – You can generate SQL from JSON, but also parse existing SQL and output a JSON representation. Helpful when you’re inheriting a database or need to align your API models.
- **Migration scripts with rollback** – It doesn’t just spit out a schema; it generates incremental migration files (with `up` and `down` steps) that you can plug into a CI/CD pipeline or version control.
- **Schema-as-code workflow** – You can keep your JSON model in Git, and the tool can diff changes to produce safe migrations.

The stack is Next.js (TypeScript) on the frontend, with a Node.js service that uses `pg` for introspection and a lightweight parser for SQL/JSON transformation. The “AI” piece is just GPT-4 via API for now—it’s not doing anything magical, just pattern-matching common structures.

I’ve been using it locally for a few months and finally put a live version here:  
[https://postgresql-schema-generator-from-json-629qn77yp.vercel.app](https://postgresql-schema-generator-from-json-629qn77yp.vercel.app)

It’s still early, but it’s saved me hours of manual typing and debugging. If you work with Postgres and JSON often, I’d love for you to try it and give **brutal feedback**. Does the output look sane? Are there edge cases I’ve missed? Would you use something like this, or does it feel like overkill?

# ShipMicro V15 System Review Roundtable

### Architect (Gemini - 架构师)
**最致命缺陷：技能层与执行层存在循环依赖死锁风险**  
`agent_skills/__init__.py` 中技能模块直接调用 `app_builder` 和 `self_heal`，而执行层的 `self_heal` 在修复失败时又会触发 `agent_skills.Debugging` 技能，形成未封装的递归调用链。当连续3次编译失败时，系统会陷入无限自愈循环。

**必须修改：**  
1. 在 `self_heal.py` 第47行添加死锁检测计数器，超过阈值立即转人工干预通道。  
2. 重构 `agent_skills/core.py`，将技能调用改为异步消息队列（Celery），通过状态机明确技能生命周期。

**隐蔽漏洞：memory_bank 的轻量RAG可能污染决策**  
`memory_bank/retriever.py` 使用余弦相似度检索历史决策，但未过滤失败案例。当系统遇到相似关键词时，可能重复错误策略。

**必须修改：**  
为每条记忆记录添加 `success_score` 字段（来自 `ui_tester` 得分），检索时加权筛选，避免负面经验传播。

### Code Reviewer (DeepSeek - 研发总监)
**最致命缺陷**：`memory_bank` 的纯正则/词频检索在复杂技能复用场景下召回率严重不足，导致 `self_heal` 因检索不到有效历史解决方案而陷入“尝试-失败-重复检索”的死循环，尤其在新品类（如游戏）开发中会指数级放大。

**隐蔽风险**：`ui_tester` 的 Playwright 断言依赖静态选择器，当 `app_builder` 生成动态 DOM 结构时，测试会随机性失败，造成误判并触发不必要的自愈循环，消耗资源。

**必须立即执行的升级**：
1. **替换 `memory_bank.py` 的检索核心**：集成轻量向量数据库（推荐 `chromadb`），将历史经验（代码片段、错误解决方案）编码为嵌入向量，结合元数据过滤，提升相似案例召回准确率。
2. **硬化 `ui_tester.py` 的定位策略**：在 `run_playwright_test` 函数中增加自适应选择器回退机制（如优先使用 `data-testid`，动态生成时回退到语义化文本或角色定位），并加入失败截图与差异对比日志，避免误触发自愈。

**行动指令**：本迭代暂停新功能，优先完成上述两项升级，确保自愈引擎的决策依据和测试反馈的可靠性。

### Product Manager (Doubao - 产品负责人)
**最致命缺陷**：`analytics_tracker` 与 `ad_injector` 在流水线末端串行执行，形成 **“数据孤岛”** 与 **“体验冲突”**。  
- 追踪数据未实时反馈至 `skill_learner` 与 `daily_forge`，导致内容策略无法基于真实用户行为（如停留时长、点击热区）优化，闭环断裂。  
- 广告注入仅依赖页面内容关键词，未结合用户交互数据（如滚动深度），易破坏高参与度页面的体验，降低长期留存。

**必须立即执行的升级**：  
1. **修改 `skill_learner.py`**：增加 `analytics_feedback` 函数，将追踪数据（通过 `localStorage` 提取的点击率、滚动比）转换为技能调优信号，驱动 `daily_forge` 的评分权重调整。  
2. **重构 `ad_injector.py`**：在广告投放前调用 `ui_tester` 的交互热区数据，采用动态避让算法（如基于 Playwright 记录的视觉焦点坐标），确保广告位避开高互动区域，并优先插入在用户自然停顿点（如章节末尾）。  

**结果**：追踪与广告从“后置注入”变为 **“体验自适应层”**，直接支撑游戏线（高交互品类）的无损商业化启动。

### Growth Hacker (Qwen - 增长黑客)
**最致命缺陷**：`social_distributor.py` 仅做单向发帖，缺乏跨平台互动与流量闭环，导致社交推广孤立。`analytics_tracker.py` 采集数据未动态反馈至 `basic_seo.py`，SEO 优化仍为静态。

**必须修改**：
1. 在 `social_distributor.py` 中增加**跨平台评论监控与自动回复**逻辑，从 `memory_bank` 调用经验生成互动内容，形成流量矩阵循环。
2. 修改 `basic_seo.py` 的 `generate_meta` 函数，接入 `analytics_tracker` 的实时搜索关键词数据，动态调整页面 title 与 description，实现追踪驱动的 SEO 优化。

**行动指令**：立即在流水线“推广”阶段后新增“矩阵联动”步骤，连接社交互动与 SEO 动态调整。

### 🎯 Phase 26 决议
# 【Phase 26 最终升级任务清单】

基于三位专家的综合诊断，当前架构存在检索失效、测试误判、数据孤岛与推广闭环断裂等系统性风险。为确保自愈引擎的决策可靠性、提升产品体验并建立增长闭环，本阶段将聚焦以下 **5项最高优先级** 的代码改造任务。

## 升级任务清单

1.  **文件：`memory_bank.py`**
    **任务：重构检索核心，集成向量数据库**
    - **具体操作**：移除现有的纯正则/词频检索逻辑。集成 `chromadb` 作为轻量级向量存储。将历史经验（代码片段、错误日志、解决方案）通过 `sentence-transformers` 模型编码为嵌入向量并存储。在 `retrieve_context` 函数中，实现“向量相似度搜索 + 元数据过滤（如技能标签、错误类型）”的混合检索策略，显著提升复杂场景下的案例召回率与准确性，为 `self_heal` 提供可靠决策依据。

2.  **文件：`ui_tester.py`**
    **任务：硬化测试定位策略与诊断能力**
    - **具体操作**：在 `run_playwright_test` 函数中，实现**自适应选择器回退链**。定位策略优先级为：`data-testid` > `role` + `name` > 文本内容匹配。当主要定位器失效时自动尝试下一级策略。同时，为每次测试失败增加**自动截图**和**DOM结构差异对比日志**功能，精确区分是真实缺陷还是动态UI导致的误判，避免无效触发 `self_heal` 循环。

3.  **文件：`skill_learner.py` & `analytics_tracker.py`**
    **任务：建立分析数据至技能优化的实时反馈环**
    - **具体操作**：在 `skill_learner.py` 中创建 `analytics_feedback` 函数。该函数从 `analytics_tracker` 维护的 `localStorage` 或日志中，提取关键用户行为指标（如页面停留时长、元素点击率、滚动深度比）。将这些指标转换为技能调优信号（例如，为高参与度内容模板增加 `daily_forge` 的评分权重），并调用 `daily_forge` 的接口动态调整内容生成策略，实现数据驱动的闭环优化。

4.  **文件：`ad_injector.py` & `ui_tester.py`**
    **任务：实现基于交互热区的智能广告避让算法**
    - **具体操作**：重构 `ad_injector.py` 的广告位决策逻辑。在注入广告前，调用 `ui_tester` 模块中记录的**用户交互热区数据**（通过Playwright收集的点击坐标、视觉焦点区域）。实现一个动态避让算法，确保广告位（如悬浮框、插屏广告）避开高互动区域（如核心按钮、高频点击区）。算法应优先将广告插入用户自然停顿点（如长文章章节末尾、视频播放完毕时），平衡商业化与用户体验。

5.  **文件：`social_distributor.py` & `basic_seo.py`**
    **任务：构建社交-SEO流量矩阵联动**
    - **具体操作**：
        - 在 `social_distributor.py` 中增加 `monitor_and_engage` 函数。该函数监控主要社交平台帖子的评论与提及，从 `memory_bank` 检索相关经验与话术，生成并发布自动回复，将单向发帖升级为互动式流量运营。
        - 修改 `basic_seo.py` 中的 `generate_meta` 函数。使其接入 `analytics_tracker` 采集的**实时搜索关键词与排名数据**，动态调整生成页面的 `<title>` 和 `<meta description>` 标签，实现基于真实搜索流量的SEO优化，形成“推广-互动-分析-优化”的完整增长闭环。
